Detailed Findings

AUTH-Q3: Action-Level Authorization Medium

Finding
All authenticated users share a single IAM role with identical API access. No action-level differentiation at the API surface.
Gap
Cannot restrict agent to read operations while allowing humans full CRUD access.
Recommendation
Configure Cognito User Pool groups and map them to separate IAM roles via Identity Pool role mapping rules. Apply method-level authorization in API Gateway.
Effort: MediumFile: source/lib/cdk-textract-stack.ts

AUTH-Q4: Identity Propagation and Delegation Medium

Finding
API Gateway extracts Cognito JWT but Lambda functions do not propagate caller identity to downstream service calls.
Gap
No identity propagation through internal service calls. Cannot distinguish agent-initiated vs human-initiated operations.
Recommendation
Extract sub and email claims from Cognito JWT in API processor Lambda and propagate as metadata through SQS messages and DynamoDB records.
Effort: MediumFile: source/lambda/apiprocessor/lambda_function.py

AUTH-Q6: Immutable Audit Logging Medium

Finding

Gap
Logs are not immutable or tamper-evident. No CloudTrail trail in IaC.
Recommendation
Add CloudTrail trail to CDK stack with log file validation and S3 delivery with object lock.
